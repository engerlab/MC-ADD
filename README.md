::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::

Monte Carlo simulations are widely used for detector development, optimization, and performance characterization. Geant4 provides a powerful and flexible particle-transport toolkit however, its complexity can represent a significant barrier for most users and generating an executable user code could be a time-consuming task. Although existing Geant4-based frameworks provide powerful simulation capabilities, preparing detector development studies often requires a substantial amount of time to adapt existing applications or develop new simulation code before the detector itself can be investigated. To address this challenge, we developed a Monte Carlo Accessible Detector Development framework: MC-ADD, a Geant4-based framework that simplifies simulation setup through a graphical user interface, and a detector-response analysis through a dedicated post-processing script. MC—ADD is a reproducible and accessible tool that enables users to rapidly construct detector models, perform simulations, and analyze detector response with minimal simulation setup would be beneficial for detector development studies. At the same time, preserving direct access to the underlying Geant4 source code remains desirable for users whose simulation requirements evolve beyond the capabilities of predefined workflows. By combining ease of use with full access to the underlying Geant4 source code, MC-ADD provides a practical platform for detector development.


:::::::::::::::::::::::::::::::::::::
:::  Description of the framework  ::
:::::::::::::::::::::::::::::::::::::

MC-ADD has three main components:
    - A Graphical User Interface (GUI): Executed through an in-house Python script (no extra dependencies/libraries are needed) through which the user can define the radiation source and detector geometry. After geometry definition, the script automatically generates the Geant4- C++ Detector Construction class and the macro file containing all the user’s geometries. The GUI also allows the user to import mesh files (STL, OBJ files). 
    - Geant4 source code: The main source code to execute the simulation
    - Analysis Script: an in-house Python/MATLAB script to process and analyze the output files generated by the Geant4 source code.


::::::::::::::::::::::
:::  Requirements  :::
::::::::::::::::::::::

    a) Install Geant4 on your computer. MC-ADD is currently compatible with Geant4 version 11.2.2. To install Geant4, it is recommended to follow the instructions provided in the official Geant4 documentation: https://geant4.web.cern.ch/docs/
    b) Additional audiovisual resources are also available for installing Geant4, along with detailed explanations of the functionality of the different Geant4 classes: https://www.youtube.com/watch?v=eG1cXevs0Gs&list=PLLybgCU6QCGUYAOwtyI4U8cRL6ig0p3c7
    c) Install Python 3. MC-ADD was developed using Python version 3.12.5.
    d) Install the following Python packages: NumPy, Matplotlib, and Pandas.
    e) Once these dependencies have been installed, download the MC-ADD framework from this repository.

    
::::::::::::::::::::::::::
:::  Executing MC-ADD  :::
::::::::::::::::::::::::::


    :::::: Geometry Definition ::::::

    + Once MC-ADD has been downloaded to the user's preferred directory, launch the `MC-ADD_macOS_Sequoia15.py` or `MC-ADD_ubuntu24.py` Python script (depending on the user's operating system) from the terminal. To do so:
        1) Open a terminal and navigate to the directory containing the MC-ADD framework. For example:
            
            > UserTerminal% cd /user/Folder1/Folder2/MC-ADD

        2) Once inside the `MC-ADD` directory, launch the GUI by running:
            
            > UserTerminal% python3 MC-ADD_macOS_Sequoia15.py or > UserTerminal% python3 MC-ADD_ubuntu24.py
      
           Running either command will automatically open the graphical user interface (GUI) shown in `GUI.png`, which is included in this GitHub repository.



    + The GUI allows the user to define the geometric, spatial, and material properties of the radiation source and detector, as well as the radionuclide and the number of primary particles to be simulated.
    
    + The GUI is divided into six main sections: 'World', 'Source', 'Detector', 'Mesh Geometries', 'Geometry Name', and 'Macro File'.

        1) World: The world volume is defined as a 2 × 2 × 2 m cubic geometry centered at the coordinates (0, 0, 0).
                  - The world material can be selected from either 'Air' or 'Water' using the drop-down menu next to the 'Material' field.
                  - Although the default world size and material are predefined, both parameters can be modified by the user after the DetectorConstruction.cc class and the macro file have been generated.

        2) Source: This section allows the user to define the shape, dimensions, position, and material of the radiation source.
                   - Shape: The source geometry can be defined as either a 'Cylinder' or a 'Box', as these are the two most commonly used geometries in experimental setups.
                   - Dimensions: The source dimensions are specified using the three input fields corresponding to the length, width, and height of the volume (for a 'Box') or the inner radius, outer radius, and thickness (for a 'Cylinder'). All dimensions must be specified in millimeters (mm).
                   - Position: The source position is also specified in millimeters (mm) and corresponds to the coordinates at which the source volume will be placed within the world volume.
                   
                        !! IMPORTANT: Geant4 positions a volume using its geometric center as the reference point. For example, if the source is assigned a position of 10 mm along the X-axis, the center of the source volume will be located at X = 10 mm.
                   
                        It is recommended to place the radiation source at the coordinates (0, 0, 0) and use it as the reference point for positioning all other volumes.
                   
                   - Material: The source material can be selected from the drop-down menu, which contains an extensive list of materials available in Geant4.

        3) Detector: The detector is defined following the same procedure used for the radiation source. The user can specify the detector shape, dimensions, position, and material using the corresponding input fields and options described in the Source section.

        4) Mesh Geometries: This section allows the user to import mesh geometries in STL or OBJ format. These geometries must be created by the user using the CAD or 3D modeling software of their choice.

            - The STL or OBJ files must be saved in ASCII format and placed in the CAD_Files directory included with the MC-ADD framework.
            - To import mesh geometries, enter the file names exactly as they were saved, separated by commas. For example, suppose the directory contains three STL files named Collimator.stl, Block1.stl, and Block2.stl. In the first input field of this section, enter: Collimator, Block1, Block2 . In the second input field, specify the path to the CAD_Files directory. For example: /User/Folder1/Folder2/MC-ADD/CAD_Files
                
                !! IMPORTANT: Do not include the STL or OBJ subdirectories in the path. The path should point only to the CAD_Files directory. Otherwise, compilation and visualization errors may occur.
                !! IMPORTANT: If the simulation does not require any STL or OBJ geometries, leave both input fields blank.
       

        5) Geometry Name: This field allows the user to assign a name to the geometry that has just been defined.
                          - Once the GUI is closed, a copy of the DetectorConstruction class will be saved in the DetectorConstructionGeometries directory using the name specified by the user. For example, if the geometry is named TestGeometry, a text file named TestGeometry.txt will be created in the DetectorConstructionGeometries directory.
                          - Saving the geometry in this way allows the user to easily reuse it in future simulations by copying and pasting its contents back into the DetectorConstruction class, eliminating the need to redefine the entire geometry through the GUI.

        6) Macro file: This section allows the user to select the radioactive source from the drop-down menu. MC-ADD includes a list of the most commonly used alpha, beta, and gamma emitters. If the user wishes to simulate a particle or photon beam instead, the generated macro file can be modified accordingly by editing the appropriate Geant4 commands.
                       - The second drop-down menu in this section specifies where the radiation will be generated: within the volume, on the surface, from a point source, or as a beam.
                       - Finally, the user specifies the number of primary events to be simulated.

    + Once the geometry has been defined, click Save. This button will automatically generate the DetectorConstruction class, save a copy of it as a text file in the DetectorConstructionGeometries directory, and generate the macro file.

    + After clicking Save, a confirmation window will appear indicating that the DetectorConstruction class and macro file have been generated successfully. Click OK to close the confirmation window.

    + Click Close to exit the GUI.


    :::::: Radiation Transport Simulation ::::::

    Before compiling the source code and running the simulation, there are several important considerations to keep in mind.

    1) Adding More Materials: If materials or compounds that are not included in the Geant4 material database are required, follow the steps below: 
       a) Open the DetectorConstruction.cc file located in the src directory.
       b) Navigate to the Materials section, located around line 61.
       c) To add a new material from the Geant4 NIST database, follow the syntax used in the existing examples (e.g., WorldMat, PbMat, SrcMat, etc.), but assign a unique variable name. For example: *UniqueName = nist->...
       d) To define a new compound or custom material, navigate to the User-defined materials/compounds section, located around line 72.
       e) This section provides the required syntax and parameters for defining custom materials. The included examples are LYSO (line 84), Stainless Steel 316LVM (line 92), Pebax C2H40 (line 108), Silicone (line 144), Optical Grease (line 121), and PEEK (line 128).
       f) Follow these examples to define the desired compound.
       g) Be sure to save all changes made to the DetectorConstruction.cc file.


    2) Mesh Geometry Materials: At present, the MC-ADD GUI does not provide an option to assign materials to mesh geometries. This is intentional, as the number of mesh geometries imported by the user can vary from one simulation to another. Providing a material selection field for each imported mesh would unnecessarily clutter the GUI and reduce its usability.
    
    Moreover, the purpose of MC-ADD is not to automate every aspect of the simulation setup. Instead, MC-ADD is designed to streamline a significant portion of the workflow involved in creating a basic Geant4 simulation geometry, including the world volume, radiation source, and detector, while generating source code that is free of compilation errors. At the same time, MC-ADD allows users to access and navigate the generated source code, helping them understand how the simulation is implemented and encouraging further customization.
    
    To assign materials to mesh geometries, follow the steps below:
        a) Open the DetectorConstruction.cc file located in the src directory.
        b) Locate the section titled Mesh Geometries, beginning around line 191. This section contains instructions for adding additional mesh files (if applicable) and assigning materials to the mesh geometries (lines 195–223).
        c) Add // at the beginning of the command /* located on line 229 to enable the Mesh Geometries section.
        d) Add // at the beginning of the command /* located at the end of the Mesh Geometries section, marked with the comment // ... UP TO HERE!
        e) To assign a material to a mesh geometry, first define the desired material by following the steps described in Section 1).
        f) Once the material has been defined, locate the variable NAME_log, where NAME corresponds to the mesh geometry name specified through the GUI.
        g) Replace the variable MATERIAL with the material defined in Step 1).

        These same steps are also described in the DetectorConstruction file.


    3) Modifying the macro file: The macro file can be modified to customize the command-based scoring parameters and the type of radiation to be simulated. 
        a) To modify the voxelization, edit the values on lines 22 and 23 of the macro file.
            IMPORTANT: Do not modify the scoring mesh position defined on line 24 unless necessary. If this position is changed, the scoring mesh will no longer be aligned with the detector volume defined in DetectorConstruction.cc.
        b) To score additional quantities (e.g., dose, flux, etc.) or apply particle filters, first define the quantity to be scored, then define the corresponding particle filter, and finally close the scoring mesh using the /score/close command.
        For a complete list of available scoring quantities and particle filters, see: https://geant4-userdoc.web.cern.ch/UsersGuides/ForApplicationDeveloper/html/Detector/commandScore.html
        c) At the end of the macro file, there is a section titled Scoring Files, where the scored data are written to output files whose names are specified by the user.
        d) To simulate a particle beam instead of a radionuclide:

            - Replace ion with the particle of interest (e.g., e-, proton, gamma, etc.) on line 60.
            - Comment out line 61 by placing the # symbol at the beginning of the line.
            - Use the /gps/ commands to define the beam geometry. A detailed description of the available General Particle Source (GPS) commands can be found at:
 https://geant4-userdoc.web.cern.ch/UsersGuides/ForApplicationDeveloper/html/GettingStarted/generalParticleSource.html


    4) Compilation: Before running the simulation, the source code must be compiled. Execute the following commands in the terminal:
                        > UserTerminal% make clean
                        > UserTerminal% make -jX      where 'X' is the number of CPU cores available on your computer.

                    - If all modifications to the source code were made correctly and the changes were saved, the compilation should complete without any errors. At the end of the compilation, the terminal should display a message similar to:
                        > ... Done!
                        
                    - If this is your first time using MC-ADD, execute the following command once:
                        > UserTerminal% chmod a+x run.sh     (This command only needs to be executed once and does not need to be repeated for future)simulations.

                    - To run the simulation, execute the following command:
                        >UserTerminal% ./run.sh MC-ADD.mac

                    IMPORTANT: It is strongly recommended to visualize the geometry before running the simulation to verify that it has been modeled correctly.
    

     5) Visualization: The geometry can be visualized using the OGL viewer, which should be available after installing Geant4.
                        - Run the following command in the terminal to launch the visualization:
                                > UserTerminal% ./run.sh --vis

                        - This command will open the OGL visualization window. If desired, the simulation can then be executed from the Geant4 session terminal using the following command. For visualization purposes, it is recommended to use no more than 100 primary particles to avoid long rendering times.
                                > Session: /control/execute MC-ADD.mac

                        - To visualize the scoring meshes, enable the commands in the Command-Based Scorer Visualization section (around line 81) of the macro file and follow the instructions provided there.

        !!! IMPORTANT: If you wish to simulate a geometry defined in the past, just copy the content of its respective txt file located in the DetectorConstructionGeometries' folder, paste it on the DetectorConstruction.cc class, and modify the macro file accordingly!!!
